# MTPS — Usage Guide

## Prerequisites

- **Node.js** ≥ 20
- **npm** ≥ 10

## Setup

```bash
git clone <repo-url>
cd Webapp-meta-sistema_tarologico
npm install
```

## Environment Variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `VITE_API_BASE_URL` | No | `/tarological-system/api` | Cloudflare Worker API origin |
| `VITE_GITHUB_LLM_TOKEN` | No | — | GitHub Models personal access token for LLM narrative |
| `VITE_GITHUB_LLM_MODEL` | No | `gpt-4o-mini` | Model ID on GitHub Models |

Create a `.env.local` file in the project root:

```env
VITE_GITHUB_LLM_TOKEN=ghp_xxxxx
```

> Without a token the app still works — narratives fall back to the local engine.

## Development

```bash
npm run dev          # Start Vite dev server at localhost:5173
npm run build        # TypeScript check + production build
npm run lint         # ESLint
npm run test         # Vitest in watch mode
npx vitest run       # Single CI-friendly test run
npx vitest run --coverage   # Coverage report (engine/ thresholds enforced)
```

The app is served under the base path `/tarological-system/`.

## Using the Wizard

### 1. Configure (Home → "Inizia Lettura")

Choose spread layout, narrative style, and archetype family on the **Configura** page.

| Spread Layout | Cards | Description |
|---|---|---|
| Croce Celtica | 10 | Classic 10-position cross |
| Tre Carte | 3 | Past / Present / Future |
| Ferro di Cavallo | 7 | Seven-position horseshoe |
| Relazione | 6 | Relationship dynamics |

| Narrative Style | Tone |
|---|---|
| Formale | Academic, structured |
| Poetico | Lyrical, metaphorical |
| Analitico | Data-driven, precise |
| Mistico | Esoteric, symbolic |

### 2. Generate

Click **Genera Lettura**. The engine:

1. Draws cards for the chosen spread.
2. Runs adaptive iteration (up to 8 rounds) seeking quality Q ≥ 0.6.
3. Stops early when **ΔQi < τ** (τ = 0.02) for 2 consecutive iterations.
4. Computes D1–D6 quality dimensions and composite score.
5. Auto-runs LTL verification.

### 3. Explore Tabs

| Tab | Content |
|---|---|
| **Carte** | Visual card grid with positions and meanings |
| **Qualità** | D1–D6 radar chart, meaning radar, iteration history chart |
| **Verifica** | LTL property results (6 properties, 4 temporal classes) |
| **Narrativa** | Generated reading text |

### 4. Refine

Click **Affina Lettura** to re-run the engine with a higher quality target (+0.1, max 0.9) and tighter delta threshold (τ = 0.01). The result is only accepted if Q improves.

### 5. Narrative Options

- **Narrazione LLM** — Sends the spread to GitHub Models (gpt-4o-mini) for a rich narrative. Falls back to local engine on error.
- **Narrativa Locale** — Instant, offline narrative generated by the local engine.

The active source is shown by an icon: 🤖 (LLM) or 🖥 (Local).

### 6. Verify (standalone)

Navigate to **Verifica** in the sidebar to run LTL verification independently on any generated spread.

## Debug Panel

Press **Ctrl + Shift + D** to toggle the debug overlay, showing:

- Current application state
- Engine parameters
- Last generation/verification results

## LTL Properties Verified

| Property | Temporal Class | Description |
|---|---|---|
| Arcana Uniqueness | Safety | No duplicate Major Arcana |
| Positional Coherence | Invariant | Each card fits its spread position |
| Narrative Continuity | Liveness | Reading maintains thematic thread |
| Spread Completeness | Safety | All positions filled |
| Elemental Balance | Invariant | Suit distribution within bounds |
| Symbolic Density | Liveness | Sufficient symbolic richness |

## Deployment

### Frontend (GitHub Pages)

The CI workflow builds and deploys automatically on push to `main`:

```bash
npm run build   # Output in dist/
```

### Backend (Cloudflare Worker)

```bash
cd cloudflare-worker
npx wrangler deploy
```

Set the worker URL via `VITE_API_BASE_URL` if different from the default.

## Testing

97 tests cover the engine and tutorial modules:

- **Scoring** — D1–D6 individual dimensions, composite Q, thresholds
- **Iteration** — ΔQi tracking, convergence reasons, dimension trends, callbacks
- **LTL** — All 6 properties, 4 temporal property types, duplicate/completeness checks
- **Spreads** — All 4 layouts, positions, coordinates, card counts
- **Narrative** — All 4 styles, card references, dominant dimension synthesis
- **Meaning** — Card meanings, spread analysis, weight configurations
- **Archetypes** — Family lookups, custom fallback
- **Tutorial** — Provider state, auto-start, navigation, skip/finish, persistence, page filtering
- **InfoTooltip** — Hover/focus visibility, ARIA accessibility, custom placement, hide timer

Coverage thresholds (engine/): **90% statements, 80% branches, 85% functions, 90% lines**.

## Interactive Tutorial & Tooltips

### Product Tour

The app includes a built-in guided tour that highlights key UI elements step-by-step.

**Auto-start:** The tour fires automatically on the first visit (1.2 s delay). Once completed or skipped, it won't appear again (persisted in `localStorage` under `mtps-tutorial`).

**Manual trigger:** Click the **?** (HelpCircle) button in the header to start a page-specific tour for the current route.

### Tour Sequence per Page

| Page | Steps | Targets |
|---|---|---|
| **Welcome** | Hero intro, Begin button, Feature cards | `tour-welcome-hero`, `tour-welcome-begin`, `tour-welcome-features` |
| **Configure** | Archetype family, Step indicator, Next/Generate btn | `tour-config-archetype`, `tour-config-step-indicator`, `tour-config-next` |
| **Generate** | Generate button, View tabs, Action bar | `tour-gen-generate-btn`, `tour-gen-view-tabs`, `tour-gen-actions` |
| **Verify** | Info panel, Run button, Results area | `tour-verify-info`, `tour-verify-run`, `tour-verify-results` |

### Navigation

- **Avanti** — Next step
- **Indietro** — Previous step
- **Salta Tutorial** / **✕** — Skip and close

### Info Tooltips

Key labels display an ℹ️ icon on hover/focus showing a concise explanation:

- **Archetype Family** — Mapping carta→archetipo
- **Spread Layout** — Disposizione spaziale ρ
- **Narrative Style** — Tono narrativo
- **Meaning Weights** — Pesi semantici μ(c, θ)
- **D1-D6 Quality** — Dimensioni di qualità
- **Iteration History** — ΔQi < τ convergence
- **LTL Verification** — 6 proprietà / 4 classi temporali

Tooltips are accessible: they respond to keyboard focus and use `aria-describedby`.

### Adding New Tour Steps

1. Open `src/context/TutorialContext.tsx`
2. Add a `TourStep` entry to the `TOUR_STEPS` array with `id`, `target` (element ID), `title`, `content`, `placement`, and `route`
3. Add the matching `id` attribute to the target JSX element in the page component
4. The step will automatically appear in the page tour

### Resetting the Tutorial

Clear the `mtps-tutorial` key from `localStorage`, or call `resetTutorial()` from the `useTutorial()` hook.
